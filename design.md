## 1. 设计定位

本系统采用“深色科技风”作为统一视觉方向。

目标效果：

- 看起来像 AI 视频监控平台。
- 适合课程答辩现场展示。
- 页面层级清楚，重点信息醒目。
- 不追求复杂动效，优先保证稳定、清晰、可实现。

关键词：

```text
深色背景 / 荧光蓝绿 / 监控面板 / 数据可视化 / 告警高亮 / 工程可实现
```

## 2. 整体视觉风格

### 2.1 风格方向

| 项目 | 设计要求 |
|---|---|
| 主基调 | 深色科技风 |
| 使用场景 | AI 安防监控、视频分析、数据看板 |
| 页面气质 | 稳定、专业、偏工程化 |
| 视觉重点 | 视频画面、告警事件、系统状态、统计数据 |
| 避免风格 | 过度花哨、低对比度、浅色后台模板感 |

### 2.2 推荐观感

页面应类似“监控中心大屏 + 管理后台”的结合：

- 左侧为稳定导航。
- 顶部显示系统状态。
- 主区域使用卡片承载内容。
- 重要状态用颜色区分。
- 告警内容使用红橙色强化。

| 状态 | 颜色 | 使用位置 |

| 在线 / 正常 | `#00FFB2` | 后端、数据库、算法正常 |

## 7. 8个页面视觉设计

### 7.1 登录页 `/login`

设计目标：

- 一眼看出是 AI 安防系统。
- 页面简洁，突出登录入口。

布局：

```text
深色渐变背景
左侧：系统标题 + 简短说明 + 科技线条装饰
右侧：登录卡片
```

登录卡片内容：

- 标题：管理员登录
- 账号输入框：`username`
- 密码输入框：`password`
- 登录按钮
- 错误提示

视觉重点：

- 登录卡片使用半透明深色背景。
- 卡片边框使用荧光蓝。
- 登录按钮使用主色 `#00D8FF`。

### 7.2 系统首页 `/`

设计目标：

- 作为系统总览。
- 展示“系统已启动、模块正常、数据可用”。

页面结构：

```text
第一行：后端状态 / 算法状态 / 数据库状态 / 设备状态
第二行：今日事件 / 累计事件 / 未确认事件
第三行：最近事件列表
```

6月25日实现策略：

- 先接入 `GET /api/health`。
- 接入 `GET /api/system/status`。
- 统计卡片和最近事件列表显示静态占位。

6月30日实现策略：

- 接入 `GET /api/statistics/overview`。
- 替换静态占位为真实统计数据。

### 7.3 视频分析页 `/analysis`

设计目标：

- 作为主演示核心页面。
- 让老师清楚看到“上传、ROI、分析、报警结果”。

页面结构：

```text
左上：视频上传卡片
右上：快速参数卡片，仅放检测置信度
中间：视频首帧 + ROI 绘制区
下方：任务进度条
底部左：结果视频
底部右：事件列表
```

视觉重点：

- ROI 使用荧光蓝边框。
- 目标检测框使用绿色。
- 报警事件使用红色。
- 进度条使用蓝色渐变。

说明：

- 本页只保留检测置信度快速调整。
- 完整 6 个系统参数放在 `/settings` 页面，避免两处不同步。

### 7.4 报警中心页 `/alarms`

设计目标：

- 新事件必须醒目。
- 确认、误报操作要清楚。

页面结构：

```text
顶部：最新报警大卡片
中部：报警事件表格
底部：事件截图 + 事件详情
```

视觉重点：

- 未确认报警使用红橙色边框。
- 弹窗可使用轻微闪烁边框，但不要高频闪动。
- 确认按钮为绿色，误报按钮为灰色。

### 7.5 历史事件页 `/history`

设计目标：

- 快速定位历史事件。
- 方便查看截图和视频证据。

页面结构：

```text
顶部：筛选条件
中部：事件表格
底部左：截图预览
底部右：结果视频播放器
```

视觉重点：

- 筛选区保持紧凑。
- 表格突出时间、置信度、状态。
- 视频播放器区域用深色背景。

### 7.6 数据看板页 `/dashboard`

设计目标：

- 为 PPT 和答辩提供视觉亮点。
- 展示系统有统计分析能力。

页面结构：

```text
第一行：今日事件 / 累计事件 / 未确认事件
第二行：事件趋势折线图
第三行：置信度分布柱状图 + 状态分布饼图
底部：最近事件 Top5
```

ECharts 配色：

| 图表 | 主色 |
|---|---|
| 折线图 | `#00D8FF` |
| 柱状图 | `#00FFB2` |
| 饼图 | 蓝 / 绿 / 黄 / 红 |

图表背景透明，外层使用深色卡片。

### 7.7 参数设置页 `/settings`

设计目标：

- 参数清晰，不复杂。
- 让用户知道这些参数影响后续任务。

页面结构：

```text
一个大表单卡片
检测置信度
连续下降帧占比
最小垂直位移
最小连续帧数
ROI 内轨迹点占比
报警冷却时间
保存按钮 / 恢复默认按钮
```

视觉重点：

- 使用分组标题“检测参数”“报警参数”。
- 输入框右侧显示单位，如 px、帧、秒。
- 保存成功使用绿色提示。

### 7.8 实时监控页 `/monitor`

设计目标：

- 展示摄像头接入能力。
- 作为辅演示，不影响主流程。

页面结构：

```text
顶部：摄像头状态卡片
中间：MJPEG 实时画面
右侧或底部：FPS、分辨率、摄像头状态
```

视觉重点：

- 视频区域使用黑色面板。
- 检测框为绿色。
- 摄像头运行中显示蓝色状态点。
- 摄像头异常显示红色提示。

## 8. 动效规范

动效只做辅助，不影响稳定性。

允许使用：

- 卡片 hover 轻微上浮。
- 按钮 hover 颜色变化。
- 页面加载时淡入。
- 报警卡片边框轻微闪烁。

避免使用：

- 大面积复杂动画。
- 长时间循环背景动画。
- 影响低配电脑性能的粒子效果。

## 9. 空状态与异常状态

### 9.1 空状态

| 场景 | 展示文案 |
|---|---|
| 无事件 | 暂无报警事件 |
| 无统计数据 | 暂无统计数据 |
| 无截图 | 暂无截图 |
| 无结果视频 | 结果视频生成中 |

### 9.2 异常状态

| 场景 | 展示方式 |
|---|---|
| 后端离线 | 红色状态卡片 |
| token 失效 | 跳转登录页 |
| 上传失败 | 页面顶部错误提示 |
| 分析失败 | 进度条变红，显示错误原因 |
| 摄像头不可用 | 红色提示卡片 |

## 12. 交付要求

前端实现时应满足：

- 8 个页面视觉风格统一。
- 深色背景下文字对比度清晰。
- P0 页面优先完成视觉一致性。
- 主演示路径页面不能出现明显错位。
- 空状态和错误状态必须有提示。
- 答辩截图应优先截取登录页、首页、视频分析页、报警中心、历史事件页、数据看板页。

## 13. 结论

本设计规范确定了高空抛物监测系统前端的深色科技风视觉方向。

刘康后续开发页面时，应以本文档作为样式基准，以《刘康6月24日前端路由与页面布局草案》作为布局基准，以《详细API设计》作为接口基准。
