# 罗龙飞负责人任务清单

## 0. 文档口径

本清单是《详细开发计划》的负责人拆解版，只写罗龙飞个人需要盯住、协调、验收和交付的内容。

执行优先级：

1. 每日安排以 [详细开发计划.md](详细开发计划.md) 为准。
2. 功能、数据表、接口范围以 [详细需求分析.md](详细需求分析.md) 为准。
3. 项目背景、演示目标和交付物以 [高空抛物监测系统-前期规划.md](高空抛物监测系统-前期规划.md) 为参考。

说明：前期规划中的旧排期和《详细开发计划》不一致时，以《详细开发计划》为准。例如 6月27日按《详细开发计划》处理为完全休息日。

## 1. 负责人定位

罗龙飞在本项目中不是单独做某一个小模块，而是负责把项目从“能做”推进到“能演示、能验收、能答辩”。

核心职责：

- 冻结需求边界，避免成员各做各的。
- 统一接口、数据表、算法输入输出格式。
- 推进主演示闭环：登录 → 上传视频 → ROI → YOLOv11检测 → DeepSORT跟踪 → 轨迹判断 → 报警 → 入库 → 历史回放 → 数据看板。
- 做前后端、算法、测试之间的集成协调。
- 控制项目进度、风险和最终答辩效果。
- 负责 PPT 主线、演示脚本、现场部署和答辩讲解。

接口范围已经包含在《详细需求分析》第 10 节。本节不是重新定义需求，而是作为联调检查清单。后续如果要开发，应单独补《详细API设计》，写清 URL、请求参数、响应 JSON、错误码和示例。

### 13.1 前后端接口

- `POST /api/auth/login`
- `GET /api/health`
- `GET /api/system/status`
- `POST /api/videos/upload`
- `GET /api/tasks/{id}`
- `GET /api/events`
- `GET /api/events/{id}`
- `PATCH /api/events/{id}/status`
- `GET /api/statistics/overview`
- `GET /api/settings`
- `PUT /api/settings`
- `POST /api/camera/start`
- `POST /api/camera/stop`
- `GET /api/camera/stream`

### 13.2 后端调用算法接口

必须统一为：

```python
run_video_analysis(
    video_path,
    output_dir,
    roi,
    settings
)
```

建议返回：

```python
{
    "status": "success",
    "total_frames": 300,
    "processed_frames": 300,
    "result_video_path": "outputs/result.mp4",
    "events": [
        {
            "track_id": 1,
            "confidence": 0.82,
            "snapshot_path": "events/snapshots/1.jpg",
            "trajectory": [
                {"frame_id": 1, "timestamp": 0.04, "x": 100, "y": 80}
            ]
        }
    ]
}
```

## 14. 你要重点检查的风险

| 风险 | 你要做的动作 |
|---|---|
| 算法效果差 | 提前选主演示视频，允许预训练模型 + 规则判断 |

| 前后端字段不一致 | 每天 18:00 接口同步 |
| 成员 Git 不熟 | 统一分支，禁止直接改 main |
| 文档证据不足 | 每天收集截图、日志、测试记录 |
| 现场网络异常 | 所有模型、视频、依赖、本地数据库提前准备 |
| 主演示机故障 | 陈磊 RTX3050 作为备机 |
