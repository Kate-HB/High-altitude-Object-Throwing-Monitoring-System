## 完成结果

### 1. 文件服务端点
- **问题**：前端 `GET /api/files?path=...` 获取结果视频/快照，后端无此路由
- **修复**：新建 `backend/app/api/files.py`，支持路径越界保护，在 `main.py` 注册

### 2. 实时进度
- **问题**：pipeline子进程只在结束时写DB，`processed_frames` 全程为0
- **修复**：pipeline每帧写 `outputs/task_N/progress.json`，`task_service.get_task()` 运行时读取覆盖 `processed_frames`

### 3. ROI检测级过滤
- **问题**：ROI只在行为分析第5条件生效，框外目标仍被检测和跟踪
- **修复**：`pipeline.py` 检测后立即过滤——检测框中心不在ROI内则丢弃

### 4. 行为分析斜率法
- **问题**：帧间检测框抖动导致逐帧 `downward_ratio ≈ 0`，所有轨迹event=0
- **根因**：`y_i > y_{i-1}` 逐帧比较被检测噪声淹没（track 75: downward_ratio=0.047，实际整体下降54.7px）
- **修复**：改用线性回归斜率判定下降趋势，抗帧间噪声
- **验证**：task_49: 2232帧 → events=2

### 5. 快照画框
- **问题**：快照在绘制前保存，只有原始帧无标注
- **修复**：重构循环顺序——先评估收集触发列表→绘制检测框/轨迹/ROI→再保存快照。触发目标加红框+EVENT ID标签

### 6. 前端快照预览
- 事件列表自动加载快照缩略图（通过 `/api/files`）
- 点击overlay弹窗放大，点背景关闭

### 7. 前端页面保活

## 技术决策

- 视频编码：放弃FFMPEG，统一用Windows Media Foundation (MSMF) + H264 FourCC
- 行为判断：放弃逐帧下降率，改用线性回归斜率（dy/dt > 0 = 下降），更抗噪
- 环境隔离：后端.venv (FastAPI) ↔ Conda base (torch) 通过 subprocess + pipeline_cli.py 桥接
