## 遇到的问题

### 1. 缺少嵌入式设备和外接摄像头

项目原设想偏向边缘设备部署，但当前小组只有笔记本，缺少真实摄像头部署环境。

### 解决办法

将项目演示方案调整为“上传视频检测为主，笔记本摄像头实时检测为辅”。这样既能展示完整流程，又能降低硬件依赖，适合课程答辩场景。

### 2. 数据集获取和训练成本较高

高空抛物相关数据集较少，且完整训练 YOLO 模型需要较长时间和较高 GPU 资源。

### 解决办法

采用公开数据集和预训练模型作为基础，不把从零训练作为主要目标。项目重点放在系统流程、检测效果展示、轨迹分析和报警闭环上。

### 3. 小组成员 Git 使用经验不足

多数成员只把 Git 当作上传和下载工具，不熟悉分支协作、提交规范和冲突处理。

### 解决办法

编写 GitHub 团队协作指南，规定统一分支流程、提交格式和 Pull Request 规则，减少协作混乱。

### 4. 模块较多，容易出现职责不清

项目包含前端、后端、算法、数据、文档和测试，若没有边界，后期容易重复开发或遗漏功能。

### 解决办法

先搭建清晰的项目框架，将系统拆分为 frontend、backend、algorithm、docs、tests、scripts 等目录，为后续成员分工开发提供统一入口。
