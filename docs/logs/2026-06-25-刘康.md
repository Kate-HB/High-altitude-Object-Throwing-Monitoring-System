## 遇到的问题

### 问题1：登录成功后被立即踢回登录页
**原因**：后端返回的token_expire可能是秒级Unix时间戳，与 `Date.now()`（毫秒）比较恒为假。
**修复**：统一使用 `Date.now() + 3600000`（1小时）作为过期时间，不受后端格式影响。

### 问题2：登录失败不显示错误信息
**原因**：后端登录失败返回 HTTP 200 + `{code: 401, detail: "账号或密码错误"}`，axios不会进catch。
**修复**：在try内先判断 `res.data.code`，非200手动设置errorMsg。

### 问题3：前端取值层级错误
**原因**：后端所有接口统一返回 `{code: 200, data: {...}, message: "success"}`，token实际在 `res.data.data.token`，不是 `res.data.token`。
**修复**：在响应拦截器中统一解包，将 `body.data` 提升到 `res.data`，后续所有接口调用无需手动取 `.data.data`。

### 问题4：health接口格式特殊
**原因**：`GET /api/health` 返回混合结构 `{status: "ok", code: 200, data: {...}}`，`fetchHealth()` 用的原生fetch不走axios拦截器。
**处理**：health.js保持原生fetch，HomeView中直接取 `h.status`，与axios调用路径分离。

## 解决办法

核心策略：**响应拦截器统一解包**。在axios interceptor中一次性处理 `{code, data, message}` 格式，所有页面组件无需关心后端包装层。特殊情况（health用原生fetch、FastAPI HTTPException用detail字段）单独处理。
