### 数据库 events 表

| 字段 | 类型 | 来源 |
|---|---|---|
| id | INTEGER PK | 自增 |
| video_task_id | INTEGER FK | 后端写入 |

| confidence | REAL | 算法 EventInfo.confidence |
| status | TEXT (unconfirmed/confirmed/false_alarm) | 默认 unconfirmed |

### 前后端事件接口对齐 ✅

| 接口 | 后端 | 前端调用 | 字段对齐 |
|---|---|---|---|
| 事件列表 | `GET /events?task_id=&status=&limit=&offset=` | `fetchEvents(params)` | ✅ |
| 事件详情 | `GET /events/{id}` | `fetchEvent(id)` | ✅ |
| 状态更新 | `PATCH /events/{id}/status` body: `{status}` | `updateEventStatus(id, status)` | ✅ |

响应格式统一为 `{code, data, message}`，前端 `request.js` 拦截器自动解包 `data` 字段。
