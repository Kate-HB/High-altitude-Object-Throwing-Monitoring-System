## 三、接口依赖

所有接口基础路径：`http://127.0.0.1:8000/api`

| 页面 | 依赖接口 | 说明 |
|---|---|---|
| 登录页 | `POST /api/auth/login` | 获取token，存localStorage |
| 系统首页 | `GET /api/health`、`GET /api/system/status`、`GET /api/statistics/overview` | 服务状态+核心统计 |
| 视频分析页 | `POST /api/videos/upload`、`POST /api/tasks/{id}/analyze`、`GET /api/tasks/{id}`、`GET /api/files?path=outputs/...` | 上传→ROI→分析→进度轮询→结果（files需带 `?path=` 查询参数） |
| 实时监控页 | `POST /api/camera/start`、`GET /api/camera/status`、`GET /api/camera/stream`、`POST /api/camera/stop` | MJPEG流，`<img>`标签直显 |
| 报警中心页 | `GET /api/events`、`GET /api/events/{id}`、`PATCH /api/events/{id}/status` | 事件列表+详情+确认/误报 |
| 历史事件页 | `GET /api/events`（筛选）、`GET /api/events/{id}`、`GET /api/files?path=outputs/...` + `?path=events/snapshots/...` | 时间/状态/置信度筛选，结果视频和截图通过files接口获取 |
| 数据看板页 | `GET /api/statistics/overview` | 今日报警、累计事件、趋势图、分布图 |
| 参数设置页 | `GET /api/settings`、`PUT /api/settings` | 6个参数读写+校验 |

**鉴权规则**：
- 登录、健康检查、MJPEG流：不需要token
- 其他所有接口：`Authorization: Bearer <token>`
- 401响应 → 跳转登录页

**路由守卫设计**（`router/index.js`）：
- `router.beforeEach` 拦截导航，检查 `localStorage` 中是否存在未过期的token
- 无token访问受保护页面 → 重定向 `/login`
- 已登录访问 `/login` → 重定向 `/`（避免重复登录）
- token过期判断：登录时存储 `expire` 时间戳，守卫中对比 `Date.now()`
- axios拦截器作为第二道防线：响应401时清除token并跳转登录页

## 九、边缘情况处理清单

| 场景 | 处理方式 |
|---|---|
| 后端离线 | 首页显示"后端服务离线"，红色指示灯 |
| 登录失败 | 表单下方显示错误信息"账号或密码错误" |
| Token过期 | 拦截401响应，清除token，跳转登录页 |
| 上传非视频文件 | 前端校验文件扩展名（.mp4/.avi/.mov/.mkv），不支持的格式弹出提示"请选择MP4/AVI/MOV/MKV格式视频" |
| 视频上传超时 | 显示"上传超时，请重试" |
| 任务分析失败 | 进度条变红，显示error_message |
| 事件列表为空 | 显示"暂无报警事件"空状态插画 |
| 截图不存在 | 显示占位图"暂无截图" |
| 结果视频不存在 | 禁用回放按钮，提示"结果视频生成中" |
| 统计加载失败 | 卡片显示"--"，图表区域显示"数据加载失败" |
| 摄像头被占用 | 显示"摄像头不可用，请检查设备" |
| 参数校验失败 | 表单字段标红，显示校验提示 |
