# 推理逻辑说明

## 整体架构

```
摄像头/视频帧 → YOLOv11检测 → IOU多目标跟踪 → 6条件行为判定 → 事件输出
                                                      ↓
                                              标注帧（框+轨迹+事件）
```

**两种应用场景共用同一套算法模块：**

| 场景 | 入口 | 说明 |
|---|---|---|
| 视频文件分析 | `algorithm/pipeline.py` | 离线处理，输出 result.mp4 |
| 摄像头实时AI | `backend/app/services/camera_ai.py` | 在线处理，MJPEG 推流 |

---

## 一、YOLOv11 目标检测

**文件：** [`algorithm/detection/detector.py`](algorithm/detection/detector.py)

### 作用
逐帧检测画面中的"抛物"目标，输出每个目标的边界框坐标和置信度。

### 类：`Detector`

#### 初始化参数

| 参数 | 默认值 | 作用 |
|---|---|---|
| `model_path` | `models/best.pt` | YOLOv11 权重文件路径 |
| `conf` | `0.35` | 置信度阈值，低于此值的目标被过滤 |
| `device` | `"0"` | 推理设备，`"0"`=GPU0，`"cpu"`=CPU |
| `imgsz` | `960` | 推理分辨率，实际帧会被缩放到此尺寸 |

#### 输出字段

每帧返回 `DetectionInfo` 列表，每个元素含：

| 字段 | 类型 | 说明 |
|---|---|---|
| `frame_id` | int | 帧序号 |
| `bbox_x` | float | 边界框左上角 x |
| `bbox_y` | float | 边界框左上角 y |
| `bbox_width` | float | 边界框宽度 |
| `bbox_height` | float | 边界框高度 |
| `confidence` | float | 置信度 (0~1) |
| `class_name` | str | 固定为 `"falling_object"` |

#### 关键细节
- 调用 `ultralytics.YOLO.predict()` 单帧模式
- `verbose=False` 不输出日志
- `stream=False` 返回完整结果列表
- 将 YOLO 的 `xyxy` 格式转换为 `(x, y, w, h)` 格式

## 二、DeepSORT 多目标跟踪

**文件：** [`algorithm/tracking/tracker.py`](algorithm/tracking/tracker.py)

### 作用
关联相邻帧的检测结果，为每个目标分配稳定 ID，记录运动轨迹。

### 类：`DeepSORTTracker`

#### 初始化参数

| 参数 | 默认值 | 作用 |
|---|---|---|
| `max_lost` | `30` | 丢失帧数上限，超出的目标被删除 |
| `min_hits` | `3` | 最少命中帧数，达到后才视为"确认"目标 |
| `mahalanobis_threshold` | `9.4877` | 马氏距离卡方门限（4自由度95%置信） |
| `iou_threshold` | `0.3` | IOU 匹配阈值（回退用） |

#### 算法流程

```
1. Kalman预测：对每个已存在track用8维Kalman滤波器（cx,cy,a,h,vx,vy,va,vh）预测下一帧位置
2. 级联匹配（马氏距离）：按 time_since_update 从小到大，用匈牙利算法将确认track与检测结果最优匹配
3. IOU回退匹配：未匹配的暂定track和剩余确认track，用匈牙利算法+IOU代价矩阵二次匹配
4. 匹配成功 → Kalman更新（校正预测，更新bbox、中心点、轨迹、置信度、time_since_update=0）
5. 未匹配的检测 → 创建新track（Kalman初始化，hits=1）
6. 未匹配的track → time_since_update累计，超过max_lost则删除
```

#### 状态字段（`self.tracks[tid]`）

| 字段 | 说明 |
|---|---|
| `bbox` | 当前边界框 `[x1, y1, x2, y2]` |
| `center` | 当前中心点 `(cx, cy)` |
| `time_since_update` | 自上次匹配后经过的帧数 |
| `mean` | Kalman状态向量 (8,) |
| `covariance` | Kalman协方差矩阵 (8,8) |
| `hits` | 累计命中帧数 |
| `trajectory` | 轨迹列表 `[(frame_id, cx, cy, timestamp), ...]` |
| `confidences` | 置信度序列 |

#### 输出字段

每帧返回 `TrackingInfo` 列表，每个元素含：

| 字段 | 说明 |
|---|---|
| `track_id` | 目标 ID |
| `frame_id` | 帧序号 |
| `timestamp` | 时间戳（秒） |
| `center_x`, `center_y` | 中心点坐标 |
| `bbox_x`, `bbox_y`, `bbox_width`, `bbox_height` | 边界框 |

## 三、6 条件行为判定

**文件：** [`algorithm/behavior/behavior.py`](algorithm/behavior/behavior.py)

### 作用
对每个活跃跟踪目标的轨迹进行分析，判断是否构成"抛物事件"。

### 类：`BehaviorAnalyzer`

#### 判定参数（默认值）

| 参数 | 默认值 | 作用 |
|---|---|---|
| `detect_confidence` | `0.35` | YOLO 检测置信度阈值 |
| `downward_ratio` | `0.7` | （预留，当前未启用） |
| `min_vertical_distance` | `40` | 轨迹 y 方向最小位移（像素），过滤微小抖动 |
| `min_track_frames` | `3` | 轨迹最少帧数，刚出现的目标不判定 |
| `roi_required_ratio` | `0.3` | 轨迹点必须在 ROI 框内的比例 |
| `alarm_cooldown_seconds` | `3` | 同 track_id 两次触发的冷却时间（秒） |
| `imgsz` | `960` | YOLO 推理分辨率 |

#### 6 个条件（全部满足才触发）

| # | 条件 | 逻辑 | 参数 |
|---|---|---|---|
| 1 | **有轨迹** | `trajectory` 非空 | — |
| 2 | **下降趋势** | 对轨迹点 `(timestamp, y)` 做线性回归，斜率 > 0（图像坐标 y 向下增大） | — |
| 3 | **垂直位移足够** | `max(y) - min(y)` ≥ 阈值 | `min_vertical_distance` |
| 4 | **持续帧数足够** | 轨迹长度 ≥ 阈值 | `min_track_frames` |
| 5 | **在 ROI 内** | 轨迹点落在 ROI 框内的比例 ≥ 阈值 | `roi_required_ratio` |
| 6 | **冷却时间** | 距上次同 track_id 触发 ≥ 阈值 | `alarm_cooldown_seconds` |

#### 条件 2 详解（线性回归斜率）

```python
# ts: 轨迹点的时间戳序列
# ys: 轨迹点的 y 坐标序列
mean_t = sum(ts) / n
mean_y = sum(ys) / n
num = sum((ts[i] - mean_t) * (ys[i] - mean_y) for i in range(n))
den = sum((ts[i] - mean_t) ** 2 for i in range(n))
slope = num / den        # dy/dt
# slope > 0 → 判定为下降（y 坐标增加 = 图像中向下移动）
```

#### 输出字段

触发时返回 `EventInfo`：

| 字段 | 说明 |
|---|---|
| `track_id` | 目标 ID |
| `confidence` | 该轨迹的平均置信度 |
| `snapshot_path` | 事件截图路径（由调用方填入） |
| `created_at` | 触发时间 |

## 四、摄像头实时 AI（CameraAISession）

**文件：** [`backend/app/services/camera_ai.py`](backend/app/services/camera_ai.py)

### 与视频管线的差异

| | 视频管线 (`pipeline.py`) | 实时 AI (`camera_ai.py`) |
|---|---|---|
| 输入 | mp4 文件 | 摄像头帧 (numpy array) |
| 输出 | result.mp4 | 标注后的 numpy array |
| 持久化 | 结束后批量写库 | 结束后批量写库 |
| 帧率 | 全速处理 | 受推理耗时限制（逐帧 YOLO） |
| 进度 | progress.json | API 轮询 |

### 处理流程（`process_frame`）

```
1. self._frame_count += 1
2. detector.predict_frame(frame, frame_id)   → DetectionInfo 列表
3. ROI 中心点过滤（bbox 中心必须在 ROI 框内）
4. tracker.update(dets, frame_id, timestamp) → 更新跟踪
5. 遍历 active_track_ids，对每个做 behavior.evaluate()
   → 触发则收集 EventInfo
6. _draw_frame() 标注：ROI框(蓝)、检测框(绿)、轨迹线+ID、事件框(红)
7. 已触发事件保存截图到 events/snapshots/
8. 返回 (标注帧, 新事件列表)
```
