## 1. 计划口径

- 开发周期：2026年6月24日—2026年7月3日

## 6. GitHub协作规则

- 主分支：`main`，只放稳定版本。
- 开发分支：`dev`，每日集成使用。
- 个人分支格式：`feature/姓名-模块`。
- 每人每天至少提交一次有效 commit。
- commit 格式：`类型: 描述`，例如 `feat: 实现视频上传接口`。
- 禁止直接改 `main`。
- 跨模块修改必须提前说明。
- PR 必须写清：改了什么、怎么测试、是否影响别人。
- 罗龙飞负责最终审核和合并。
