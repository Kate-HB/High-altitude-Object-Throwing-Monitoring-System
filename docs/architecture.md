# 系统架构

## 1. 架构目标

项目采用前后端分离结构，将界面、业务接口、算法流水线和数据持久化分开，便于五人并行开发。当前仓库是可运行的最小框架，不包含正式的视频检测、跟踪、报警、回放和统计业务。

## 2. 模块职责

| 模块 | 目录 | 职责 |
| --- | --- | --- |
| Vue 3 前端 | `frontend/` | 展示监控、报警、历史事件、数据看板和系统设置；通过 HTTP 调用后端 API |
| FastAPI 后端 | `backend/` | 接收前端请求，管理分析任务、事件、配置和文件；调用算法流水线；提供 API |
| 算法流水线 | `algorithm/` | 按统一接口组织目标检测、目标跟踪和行为判定，输出检测框、轨迹与事件 |
| SQLite | `backend/data/` | 后续保存管理员、任务、事件、报警配置等结构化数据 |
| 文件存储 | `uploads/`、`outputs/`、`events/` | 后续保存上传视频、分析结果、事件截图和事件片段，不提交 Git |

SQLite 适用于当前单机演示，不承担多机并发和分布式存储。视频、图片和模型等大文件不写入数据库。

## 3. 数据流

```text
视频源
  ↓
FastAPI 创建分析任务
  ↓
算法流水线
  ├─ 目标检测
  ├─ 目标跟踪
  └─ 轨迹与行为判定
  ↓
事件结果
  ├─ SQLite：事件元数据
  └─ 文件目录：截图、视频片段、输出视频
  ↓
Vue 3：监控、报警、历史记录、回放和统计展示
```

视频源后续包括预置视频、用户上传视频和笔记本摄像头。后端负责业务编排，算法模块不直接操作页面或数据库；前端不直接调用算法代码。

## 4. 当前代码边界

当前已实现：

- FastAPI 应用入口和 `GET /api/health` 健康检查。
- Vue 3 状态页及后端在线状态检查。
- `AnalysisPipeline.analyze()` 统一算法占位接口。
- 后端与算法基础测试。
- Windows 前后端启动脚本。

当前未实现：

- 视频上传、摄像头采集和 FFmpeg/OpenCV 预处理。
- YOLO 模型加载、检测训练与 ONNX 推理。
- DeepSORT 跟踪、轨迹规则和高空抛物事件判定。
- SQLite 表结构、登录、事件存储和查询。
- 报警、视频回放、ROI、参数设置和数据看板。

`AnalysisPipeline` 当前固定返回 `not_ready`，仅用于确定接口边界，不能视为算法功能完成。

## 6. 后续模块落点

- 检测：`algorithm/detection/`
- 跟踪：`algorithm/tracking/`
- 行为判定：`algorithm/behavior/`
- 后端数据模型：`backend/app/models/`
- 后端业务服务：`backend/app/services/`
- 前端通用组件：`frontend/src/components/`
- 前端页面：`frontend/src/views/`
- 自动化测试：`tests/`

业务代码应遵守“前端调用后端、后端编排算法、算法返回结构化结果”的依赖方向，避免前端直接读取数据库或算法模块直接依赖 FastAPI。
