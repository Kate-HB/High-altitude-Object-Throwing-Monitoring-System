## 二、工作模式

杨锦辉不写业务代码，工作重点是**验证和记录**：

```
功能完成 → 编写测试用例 → 执行测试 → 记录问题 → 跟踪修复 → 回归测试 → 测试报告
```

**测试环境**：
- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:5173`
- 测试工具：curl、浏览器DevTools、DB Browser for SQLite
- 测试数据：准备2-3段测试视频（正常抛物 + 非抛物干扰），存放路径 `tests/test-data/`，由杨锦辉向石义焌或罗龙飞索取

## 三、测试范围

### 3.1 接口测试（对照详细API设计.md第15节）

**认证（5项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-AUTH-001 | 正确账号登录 | 返回token |
| API-AUTH-002 | 错误密码登录 | 返回code:401 |
| API-AUTH-003 | 无token访问受保护接口 | 返回401 |
| API-AUTH-004 | 空用户名登录 | 返回422（Pydantic校验失败）或400 |
| API-AUTH-005 | 空密码登录 | 返回422（Pydantic校验失败）或400 |

**系统状态（4项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-SYS-001 | 健康检查 | 顶层 `status:"ok"`（前端兼容字段），`data.status:"running"`（统一响应业务字段），注意区分两个status |
| API-SYS-002 | 系统状态（有token） | 返回四维状态（backend/database/algorithm/device） |
| API-SYS-003 | 模型文件缺失 | algorithm返回missing_model |
| API-SYS-004 | 无token访问system/status | 返回401 |

**视频任务（7项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-TASK-001 | 上传正常视频 | 创建任务成功 |
| API-TASK-002 | 上传非视频文件 | 返回415 |
| API-TASK-003 | 提交ROI启动分析 | status变为running |
| API-TASK-004 | 重复启动同一任务 | 返回409 |
| API-TASK-005 | 查询任务进度 | 返回processed_frames和progress |
| API-TASK-006 | 算法成功完成 | status变为success |
| API-TASK-007 | 算法失败 | status变为failed，含error_message |

**事件（5项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-EVENT-001 | 查询事件列表 | 分页数据 |
| API-EVENT-002 | 查询事件详情 | 含截图、结果视频、轨迹 |
| API-EVENT-003 | 确认事件 | status变为confirmed |
| API-EVENT-004 | 标记误报 | status变为false_alarm |
| API-EVENT-005 | 非法状态值 | 返回400 |

**统计（3项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-STAT-001 | 空事件时统计 | 全部为0 |
| API-STAT-002 | 新增事件后 | 累计数增加 |
| API-STAT-003 | 更新事件状态 | 状态分布变化 |

**参数（3项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-SET-001 | 读默认参数 | 返回6个参数 |
| API-SET-002 | 保存有效参数 | 成功 |
| API-SET-003 | 保存非法参数 | 返回400 |

**文件（4项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-FILE-001 | 访问结果视频 | 可播放 |
| API-FILE-002 | 访问截图 | 可显示 |
| API-FILE-003 | 访问不存在文件 | 返回404 |
| API-FILE-004 | 非法路径 | 返回400 |

**摄像头（5项）**
| 编号 | 测试项 | 预期结果 |
|---|---|---|
| API-CAM-001 | 启动摄像头 | 返回running |
| API-CAM-002 | 查询状态 | 返回running/stopped |
| API-CAM-003 | MJPEG流 | 页面可显示 |

### 3.2 前端功能测试

| 页面 | 测试点 |
|---|---|
| 登录页 | 表单校验、错误提示、登录成功跳转、token存储 |
| 系统首页 | 四维状态正确显示、统计卡片数据正确、离线状态红色指示 |
| 视频分析页 | 上传流程、ROI绘制、进度条实时刷新、结果视频播放 |
| 报警中心页 | 弹窗动画、报警音效、确认/误报操作后状态同步 |
| 历史事件页 | 筛选条件正确过滤、分页、截图预览、视频回放 |
| 数据看板页 | 图表数据与数据库一致、趋势图正确、分布图比例正确 |
| 参数设置页 | 读默认值、修改保存、校验拦截非法输入 |
| 实时监控页 | 启停控制、MJPEG流显示、设备占用提示 |
| 全局路由守卫 | Token过期后访问受保护页面→自动跳转登录页；已登录访问/login→重定向到/ |
| ROI字段完整性 | 拖拽绘制ROI后，抓包或console验证提交的JSON字段：`{roi_x, roi_y, roi_width, roi_height}` 值完整且范围合理 |

### 3.3 端到端集成测试

| 场景 | 流程 |
|---|---|
| 主演示闭环 | 登录→上传视频→画ROI→启动分析→进度查询→报警事件→确认事件→历史回放 |
| 异常流程 | 上传非视频→错误提示；未登录→跳转登录；断网→各页面降级提示 |

| 统计准确性 | 看板统计数字与 `SELECT COUNT(*)` 结果一致 |

## 十一、风险与应对

| 风险 | 应对 |
|---|---|
| 功能未完成无法测试 | 先写测试用例，功能就绪后立即执行；用mock数据模拟边界情况 |
| 测试证据不足 | 每天截图保存关键测试结果，问题记录含截图+复现步骤 |
| 缺陷修复缓慢 | 按严重程度排序，P0每日跟进，P1/P2集中处理 |
| 跨模块问题定位困难 | 问题描述写清楚复现步骤，标注可能涉及的模块和负责人 |
| 测试时间被压缩 | 优先P0主演示闭环，P1/P2酌情缩减 |
