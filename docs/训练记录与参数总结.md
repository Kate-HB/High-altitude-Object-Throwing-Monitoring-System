## 训练记录

### 第1次：b24（基准模型）

| 参数 | 值 |
|------|-----|

| epochs | 200（实际65轮早停） |
| batch | 24 |

**最佳结果（epoch 45）**：

| P | R | mAP50 | mAP50-95 | FP | FN |
|----|----|-------|----------|-----|-----|
| 0.9347 | 0.8409 | 0.8797 | 0.3359 | 30 | 10 |

### 第2次：v2（提高分辨率+冻结+增强）

| 参数 | 值 | 改动原因 |
|------|-----|----------|
| imgsz | **960** | 小目标保留更多像素 |
| batch | **4** | 适配 4GB 显存 |
| epochs | **60** | 数据量不大，60 轮可控 |
| freeze | **10** | 冻结 backbone 防过拟合 |
| box | **10.0** | 提高定位 loss 权重（默认7.5） |
| cls | **0.8** | 提高分类 loss 权重（默认0.5），减少误检 |
| scale | **0.3** | 减少 mosaic 缩放压缩（默认0.5） |
| copy_paste | **0.3** | 小目标复制粘贴增加样本密度 |
| cos_lr | **True** | 余弦退火稳定收敛 |
| workers | 4 | |
| cache | true | |
| device | 0 | |

```powershell
$env:KMP_DUPLICATE_LIB_OK=1
yolo train model=models/yolo11n.pt data=data/falling_object/data.yaml epochs=60 batch=4 imgsz=960 device=0 workers=4 cache=true optimizer=AdamW lr0=0.001 lrf=0.01 cos_lr=True patience=20 close_mosaic=10 freeze=10 box=10.0 cls=0.8 scale=0.3 copy_paste=0.3 project=runs/train name=falling_object_yolo11n_v2
```

**最佳结果（epoch 16）**：

| P | R | mAP50 | mAP50-95 | FP | FN |
|----|----|-------|----------|-----|-----|
| 0.9062 | 0.8950 | 0.8900 | 0.4030 | 29 | 14 |

**vs v1**：Recall ↑5.4%（漏检减少），mAP50-95 ↑6.7%（定位显著改善），但 Precision ↓2.9%（误检略增）。

**新问题**：视频推理中出现"衣物误检"——建筑上晾晒的衣物被识别为坠落物。因为模型没见过挂在建筑上的静止物品作为负样本。

### 第3次：v3（加入难负样本+增大batch）

数据集新增 demo.mp4 帧（含衣物/晾晒物等难负样本）：train 764正+127负，val 185正+37负。

| 参数 | 值 | 改动原因 |
|------|-----|----------|
| batch | **8** | 数据增多可提batch |
| freeze | **5** | 减半，让模型更快适应新数据 |
| mosaic | **1.0** | 全开增强多样性 |
| workers | 4 | |
| cache | true | |
| box | 10.0 | |
| cls | 0.8 | |
| scale | 0.3 | |
| copy_paste | 0.3 | |

```powershell
$env:KMP_DUPLICATE_LIB_OK=1
yolo train model=models/yolo11n.pt data=data/falling_object/data.yaml epochs=60 batch=8 imgsz=960 device=0 workers=4 cache=true optimizer=AdamW lr0=0.001 lrf=0.01 cos_lr=True patience=20 close_mosaic=10 freeze=5 mosaic=1.0 box=10.0 cls=0.8 scale=0.3 copy_paste=0.3 project=runs/train name=falling_object_yolo11n_v3
```

**结果**：误检完全消除 ✓，但约 50% 漏检。box_loss 持续在 2.x 无法下降。

**分析**：难负样本让模型变得"太保守"，cls=0.8 分类惩罚过重，模型宁愿不框也不敢乱框。

**缓解**：推理时 conf 从 0.25 降到 0.15，漏检显著改善。模型其实检测到了但置信度被压制。

---

### 第4次：P2 定制检测头 ️（本机跑失败）

为了极致提升小目标检测，定制 yolo11n-p2.yaml：
- 新增 P2 检测头（4倍下采样 160×160），保留浅层细粒度特征
- 删除 P5 大目标检测头
- P2+P3+P4 三层检测

```powershell
yolo train model=models/yolo11n-p2.yaml data=data/falling_object/data.yaml epochs=80 batch=2 imgsz=960 device=0 workers=4 cache=true optimizer=AdamW lr0=0.001 lrf=0.01 cos_lr=True patience=20 close_mosaic=10 freeze=5 mosaic=1.0 multi_scale=True box=15.0 cls=0.5 scale=0.3 copy_paste=0.3 project=runs/train name=falling_object_yolo11n_p2
```

**失败**：epoch 37 时 `cv2.error: (-4:Insufficient memory) Failed to allocate 2764800 bytes`。

**原因**：
1. P2 头网格数量是 P3 的 4 倍，计算量暴增
2. `multi_scale=True` 将图片随机放大到 1.5×960=1440，CPU 内存不足
3. 4GB 显存无法承受

**第二次尝试**：去掉 multi_scale、workers=0、cache=false，同样跑不动。

---

### 第5次：多尺度训练 ️（本机跑失败）

回退到标准 yolo11n.pt，仅开启 `multi_scale=True` 利用尺度增强。

| 参数 | 默认值 | v1 | v2 | v3 | P2尝试 | 最终推荐 | 作用 |

| freeze | 0 | - | 10 | **5** | 5 | **5** | 防小数据过拟合 |
| box | 7.5 | - | **10** | 10 | 15 | **10-15** | 强化定位loss |

| cos_lr | False | - | **True** | True | True | **True** | 余弦退火收敛 |
| mosaic | 1.0 | - | - | **1.0** | 1.0 | **1.0** | 增强多样性 |

| patience | 100 | 20 | 20 | 20 | 20 | **20-25** | 防过拟合 |
| close_mosaic | 10 | 10 | 10 | 10 | 10 | **10** | 最后N轮关mosaic |
| epochs | 100 | 200 | 60 | 60 | 80 | **60-80** | 本数据集60轮够 |
| conf(推理) | 0.25 | 0.25 | 0.25 | **0.15** | - | **0.15-0.25** | 小目标↓conf |

### 参数改动原因速查

| 问题 | 参数调整 | 原因 |
|------|----------|------|
| 误检多（FP高） | cls↑(0.5→0.8)、加难负样本、conf↑ | 增大分类惩罚，教模型区分背景 |
| 漏检多（FN高） | cls↓(0.8→0.5)、box↑(10→15)、conf↓ | 减小分类惩罚，强化定位，降低阈值 |
| 定位差（mAP50-95低） | box↑(7.5→10→15)、imgsz↑ | 加大框回归损失权重，提高输入分辨率 |
| 小目标检测难 | imgsz↑、copy_paste↑、scale↓ | 保留像素、增加样本密度、减少压缩 |
| 1. OMP Error #15 | `$env:KMP_DUPLICATE_LIB_OK=1` | Windows下OpenMP多DLL冲突 |
| 用CPU跑 | 确保Conda环境、`device=0` | 系统Python无torch，需激活Conda |
| 4GB显存OOM | batch↓、workers↓、关multi_scale | 释放GPU/CPU内存 |
| labelImg打不开 | 复制classes.txt到labels/train | labelImg在保存目录找classes.txt |
| box_loss卡2.x | box↑(15)、conf↓(推理) | 模型定位弱，框不准导致IoU低→漏检 |

## 遇到的主要问题与解决方案

### 1. 环境问题

**OMP Error #15 崩溃**
- 现象：Windows 上训练到 cache 阶段崩溃
- 原因：PyTorch 和 OpenCV 同时加载了 `libiomp5md.dll` 的两个副本
- 解决：`$env:KMP_DUPLICATE_LIB_OK=1`

**yolo 命令用 CPU 跑**
- 现象：CUDA 可用但训练在 CPU 上
- 原因：终端未激活 Conda 环境，系统 Python 无 PyTorch
- 解决：激活 Conda base 环境，确认 `python -c "import torch; print(torch.cuda.is_available())"` 输出 True

### 2. 数据问题

**标注工具 labelImg 自动关闭**
- 原因：用 bash 后台启动时 GUI 进程被终端绑定，终端关闭则 GUI 消失
- 解决：`cmd.exe /c "start /B pythonw.exe labelImg-script.py ..."` 完全脱离终端

**labels/train 缺少 classes.txt**
- 现象：labelImg 报 `FileNotFoundError: ...labels/train/classes.txt`
- 原因：labelImg 要求保存目录下存在 classes.txt
- 解决：`cp data/falling_object/classes.txt data/falling_object/labels/train/`

**图片标签不匹配**
- 现象：训练集出现孤儿图片或孤儿标签
- 原因：手动移动/复制图片时未同步对应标签
- 解决：写脚本逐文件检查并清理，空标签→负样本，缺标签→补建或删图

### 3. 训练问题

**GPU 利用率低（~50%）**
- 原因：batch=1 太小 + workers=0 数据加载串行
- 解决：增大 batch、提高 workers、开启 cache

**val/box_loss 震荡不下降**
- 现象：mAP50-95 持续低迷（0.33），box_loss 停在 2.x
- 原因：小目标定位难、输入分辨率不够、定位 loss 权重偏低
- 解决：imgsz 640→960，box 7.5→10→15

**训练早停过快**
- 现象：epoch 16 即停止，mAP 未到峰值
- 原因：patience=20 在波动数据上触发过早
- 解决：patience 提高到 25，或用 cos_lr 减少波动

**4GB 显存限制**
- 本机 RTX 3050 Laptop 4GB 无法承载：P2 定制头、multi_scale、大 batch
- 建议迁移到 RTX 5060 Ti 8GB 继续实验

### 4. 模型效果问题

**衣物误检为坠落物**
- 现象：建筑上晾晒的衣物被检测为 falling_object
- 原因：训练集负样本只有空建筑立面，没有"挂了物品但非坠落"的难负样本
- 解决：从视频截取误检帧加入训练集，标注为空标签

**消除误检后漏检暴增**
- 现象：v3 无任何误检，但 50% 漏检
- 原因：难负样本 + cls=0.8 让模型过度保守
- 解决：推理降 conf 到 0.15，下次训练降 cls 到 0.5

**小目标 Recall 始终偏低**
- 现象：即使 mAP50 达 0.89，小纸团类仍易漏检
- 原因：4000×5000 大图中纸团仅占几十像素，imgsz=960 仍需进一步优化
- 未完成方案：P2 检测头（需更大显存）、聚类分尺度训练
