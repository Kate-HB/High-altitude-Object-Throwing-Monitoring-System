# 高空抛物监测系统前期规划

## 1. 项目定位

- 课程类型：生产实习（人工智能方向）。
- 截止日期：2026年7月3日。
- 项目目标：完成可展示、可操作、可测试的高空抛物监测闭环。
- 建设重点：PPT与现场演示优先，其次是功能完整性、测试报告和算法指标。
- 运行环境：Windows笔记本单机部署，断网可完整运行。
- 实现原则：控制算法难度，优先保证主演示稳定。

## 2. 演示场景

### 2.1 主演示

使用公开楼宇坠物视频完成：

```text
上传视频 → 选择/绘制ROI → 启动分析 → YOLO检测
→ DeepSORT跟踪 → 轨迹规则判定 → 页面弹窗和声音报警
→ 保存事件 → 历史查询 → 视频回放 → 数据看板更新
```

### 2.2 辅助演示

调用笔记本摄像头，验证实时视频采集、画面显示和检测链路。可在室内使用纸团或空塑料瓶模拟向下运动，但不将其作为真实高空场景或主要验收依据。

### 2.3 演示保障

- 准备3段主演示视频：明显坠物、弱小目标、多目标或复杂背景。
- 准备2段泛化测试视频和1段正常负样本视频。
- 模型、依赖、视频和数据库全部本地保存。
- 罗龙飞RTX3050电脑主演示，陈磊RTX3050电脑作为备机。
- 预录完整演示视频，现场异常时可立即切换。

## 3. 功能范围

### 3.1 视频采集与预处理

- 支持用户上传视频、预置演示视频和笔记本摄像头。
- 校验视频格式、大小和可读性。
- 使用OpenCV/FFmpeg完成解码、缩放、抽帧、去噪和对比度增强。
- 支持在画面中绘制监控区域ROI，仅分析楼体有效区域。

### 3.2 目标检测与跟踪

- 使用YOLO检测“坠落物”单一类别。
- 使用DeepSORT为目标分配ID并记录连续轨迹。
- 在视频上绘制检测框、目标ID、置信度和运动轨迹。

### 3.3 行为判定

- 提取目标位移、运动方向、速度和连续出现帧数。
- 目标同时满足“持续向下、位移达标、连续帧数达标”时判定为疑似高空坠物。
- 不区分人为抛出和自然坠落，系统结果统一称为“疑似高空坠物事件”。

### 3.4 实时报警

- 页面弹窗提示。
- 本地声音报警。
- 保存报警时间、截图、置信度、轨迹和视频片段。
- 设置报警冷却时间，避免同一事件重复报警。

### 3.5 数据存储与回放

- SQLite保存用户、视频源、分析任务、报警事件和参数配置。
- 本地目录保存原始视频、结果视频、事件截图和事件片段。
- 支持按时间、状态、置信度查询事件。
- 支持事件详情、截图查看和视频回放。

### 3.6 Web管理平台

1. 管理员登录：单一管理员账号。
2. 实时监控：摄像头画面、检测框、轨迹、FPS和设备状态。
3. 视频分析：上传视频、绘制ROI、查看进度和结果。
4. 报警中心：报警弹窗、声音、事件详情和确认状态。
5. 历史事件：条件筛选、截图、结果视频和事件回放。
6. 数据看板：今日报警数、累计事件数、事件趋势、时段分布、置信度分布、视频源状态。
7. 系统设置：检测置信度、最小位移、连续帧数、报警冷却时间、视频源配置。

### 3.7 性能与可用性

- 支持GPU和CPU推理，优先使用CUDA。
- 模型可导出ONNX，作为部署优化或答辩技术亮点。
- 显示任务状态、处理进度、FPS和错误信息。
- 对无效视频、分析失败、模型缺失和数据库异常给出明确提示。

## 4. 本期不实现

- 真实短信、邮件和第三方消息通知。
- 多角色权限管理。
- 多摄像头并发监控。
- Linux、边缘设备或嵌入式设备实际部署。
- 任意场景下的通用识别能力。
- 人为抛出与自然坠落的精确分类。
- 复杂深度学习行为识别模型。

### 5.1 数据流

```text
视频源
  ↓
视频读取与预处理
  ↓
ROI过滤
  ↓
YOLO单类别检测
  ↓
DeepSORT目标跟踪
  ↓
轨迹特征计算
  ↓
阈值规则判定
  ├─ 非事件：继续分析
  └─ 疑似事件：报警、截图、录像、入库
                         ↓
                Web查询、回放、统计
```

### 5.2 核心接口数据

- 视频帧：`frame_id、timestamp、source_id、image`。
- 检测结果：`bbox、confidence、class_id`。
- 跟踪结果：`track_id、bbox、trajectory`。
- 事件结果：`event_id、track_id、event_score、snapshot、video_clip、created_at`。

## 6. 数据集与训练方案

### 6.1 数据来源

- 主数据集：[FADE公开数据集](https://fadedataset.github.io/FADE.github.io/)。
- 补充数据：[阿里云天池高空抛物数据集](https://tianchi.aliyun.com/dataset/180178)。
- 仅使用许可范围内的公开数据，记录数据来源和使用说明。

### 6.2 数据处理

- 将不同物体类别统一映射为“坠落物”。
- 从视频中按间隔抽帧，避免大量重复帧。
- 按原始视频或场景划分训练集、验证集和测试集，禁止同一视频的相邻帧跨集合。
- 保留正常场景和困难负样本，用于测试误报。
- 记录数据量、分辨率、场景分布、标注格式和清洗规则。

### 6.3 训练策略

- 使用YOLO小模型预训练权重迁移学习，不从零训练。
- 石义焌RTX4060 8GB负责主要训练。
- 先用小规模数据跑通训练，再扩大数据并调参。
- 保存最佳权重、训练参数、损失曲线、PR曲线、混淆矩阵和测试指标。
- 优先保证常见楼宇坠物视频可识别，不承诺任意场景准确。

## 9. 测试规划

### 9.1 功能测试

- 管理员正确/错误登录。
- 视频上传、格式校验、分析启动、进度显示和结果播放。
- 摄像头开启、关闭和异常恢复。
- ROI绘制、修改、清除和越界处理。
- 检测框、目标ID、轨迹和FPS显示。
- 报警弹窗、声音、冷却时间和事件入库。
- 历史事件筛选、详情、截图和视频回放。
- 看板统计与数据库记录一致。
- 参数修改、保存和重新加载。

### 9.2 算法测试

- 检测指标：Precision、Recall、mAP50、mAP50-95。
- 行为指标：事件级准确率、召回率、误报率、漏报率。
- 场景测试：小目标、快速坠落、遮挡、复杂背景、低对比度和正常无坠物视频。
- 对比测试：有无预处理、有无ROI、不同置信度和轨迹阈值。

### 9.3 性能测试

- GPU和CPU下的平均FPS、单帧耗时和整段视频处理时间。
- CPU、GPU显存和内存占用。
- 上传、查询、回放等接口响应时间。
- 不同视频分辨率和时长下的处理表现。

### 9.4 稳定性与异常测试

- 连续运行2小时。
- 连续上传多段视频。
- 空文件、损坏视频、不支持格式、超大文件。
- 模型文件缺失、数据库文件异常、摄像头占用。
- 分析中刷新页面、重复点击和任务失败恢复。

### 9.5 兼容与演示测试

- 罗龙飞、石义焌、陈磊三台GPU电脑部署验证。
- Chrome和Edge浏览器验证。
- 断网运行验证。
- 主演示机故障后的备机切换。
- 完整演示流程计时和至少3次预演。

## 13. 主要风险

| 风险 | 应对措施 |
|---|---|
| 公开数据下载慢或格式不统一 | 同时获取两套数据；优先用小规模数据跑通；编写统一转换脚本 |
| 模型训练耗时或效果差 | 使用预训练小模型；先保证基线；精选主演示视频；轨迹规则辅助判定 |
| 小目标、遮挡导致漏检 | 提高推理分辨率；使用ROI；调整置信度；保留最佳测试场景 |
| 正常运动物体导致误报 | 加入方向、位移、连续帧数和报警冷却阈值 |
| 前后端等待算法接口 | 第一日冻结JSON接口；先使用模拟数据开发 |
| 最后集中集成 | 6月27日前完成首次闭环；每两日集成一次 |
| 现场演示失败 | 断网部署；主演示视频本地化；备机；预录视频 |
| 文档证据不足 | 每日保存截图、问题记录、测试结果和会议纪要 |
