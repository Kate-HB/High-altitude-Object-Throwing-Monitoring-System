## 三、数据库设计（5张表）

### 3.1 system_settings（单行配置表）

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| id | INTEGER PK | 1 | 强制单行 |
| detect_confidence | REAL | 0.35 | 检测置信度阈值 |
| downward_ratio | REAL | 0.7 | 连续下降帧占比 |
| min_vertical_distance | INTEGER | 80 | 最小垂直位移(px) |
| min_track_frames | INTEGER | 5 | 最小跟踪帧数 |
| roi_required_ratio | REAL | 0.7 | ROI内轨迹点占比 |
| alarm_cooldown_seconds | INTEGER | 10 | 报警冷却(秒) |
| updated_at | TEXT | datetime('now') | 更新时间 |

### 3.2 video_tasks（分析任务表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| source_type | TEXT | `upload` / `camera` |
| source_path | TEXT | 视频文件路径 |
| status | TEXT | `pending` / `running` / `success` / `failed` |
| total_frames | INTEGER | 总帧数 |
| processed_frames | INTEGER | 已处理帧数 |
| roi_x, roi_y, roi_width, roi_height | INTEGER | ROI区域 |
| result_video_path | TEXT | 结果视频路径 |
| error_message | TEXT | 错误信息 |
| created_at, updated_at | TEXT | 时间戳 |

### 3.3 events（报警事件表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | 自增主键 |
| video_task_id | INTEGER FK | 关联任务 |
| track_id | INTEGER | 跟踪ID |
| confidence | REAL | 置信度 |
| status | TEXT | `unconfirmed` / `confirmed` / `false_alarm` |
| snapshot_path | TEXT | 截图路径 |
| result_video_path | TEXT | 结果视频路径 |
| created_at, updated_at | TEXT | 时间戳 |

### 3.4 detection_results（检测结果表）

| 字段 | 说明 |
|---|---|
| video_task_id, frame_id | 关联任务+帧号 |
| bbox_x/y/width/height, confidence, class_name | 检测框信息（默认'falling_object'） |

> ⚠️ bbox字段类型当前为REAL，需与石义焌确认YOLO输出的坐标是整数还是浮点，决定最终用INTEGER还是REAL。

### 3.5 tracking_results（跟踪结果表）

| 字段 | 说明 |
|---|---|
| video_task_id, track_id, frame_id | 关联任务+跟踪ID+帧号 |
| timestamp, center_x/y | 时间戳+中心点 |
| bbox_x/y/width/height | 边界框 |

**关键约束**：
- 外键：`events.video_task_id → video_tasks.id`
- `system_settings` 用 `INSERT OR IGNORE` 插入默认行
- 数据库文件：`backend/data/system.db`
- 启动时自动建表，WAL模式+外键约束开启

## 四、接口清单（17个）

### 4.1 认证（1个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `POST /api/auth/login` | 管理员登录 | 否 |

请求：`{"username":"admin","password":"admin123"}`  
成功：`{code:200, data:{token, username, role}}`  
失败：`{code:401, detail:"账号或密码错误"}`  
实现：硬编码校验，`secrets.token_hex(16)` 生成token，内存字典存储，24h过期

### 4.2 系统状态（2个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `GET /api/health` | 健康检查 | 否 |
| `GET /api/system/status` | 系统状态 | 是 |

`/api/health`：返回 `{status:"running", service, version, time}`  
`/api/system/status`：返回四维状态 — backend运行状态、database连接状态（SELECT 1检测）、algorithm模型加载状态（检查models/best.pt）、device GPU/CPU信息（torch.cuda检测）

### 4.3 视频上传与任务（3个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `POST /api/videos/upload` | 上传视频+创建任务 | 是 |
| `POST /api/tasks/{task_id}/analyze` | 提交ROI+启动分析 | 是 |
| `GET /api/tasks/{task_id}` | 查询任务进度 | 是 |

上传流程：
1. 校验扩展名（.mp4/.avi/.mov/.mkv），不合法返回415
2. 生成 `{uuid}_{原名}` 保存到 `uploads/`
3. OpenCV读取总帧数
4. INSERT `video_tasks`，status=`pending`
5. 可选ROI参数一并写入

ROI提交：
1. 校验任务状态为`pending`，否则409
2. 更新ROI字段，status→`running`
3. 后台线程调用 `algorithm.pipeline.run_video_analysis()`
4. 线程内更新processed_frames，处理返回结果写入events等表

进度查询：
- 计算 `progress = processed_frames / total_frames * 100`
- 关联查询events列表

**后台线程框架**：`threading.Thread` 实现，线程内try/except捕获异常，失败时写入error_message

### 4.4 事件报警（3个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `GET /api/events` | 事件列表（分页+筛选） | 是 |
| `GET /api/events/{event_id}` | 事件详情+轨迹 | 是 |
| `PATCH /api/events/{event_id}/status` | 更新事件状态 | 是 |

列表筛选：`status`, `start_time`, `end_time`, `min_confidence`  
分页：`page`(默认1), `page_size`(默认10)  
详情：关联 `video_tasks`（源视频路径）+ `tracking_results`（轨迹点数组）  
状态更新：校验 `unconfirmed/confirmed/false_alarm`，非法返回400

### 4.5 数据看板（1个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `GET /api/statistics/overview` | 统计概览 | 是 |

返回6项：
- `today_event_count` — 当日事件数
- `total_event_count` — 累计事件数
- `recent_events` — 最近5条
- `daily_trend` — 按日期分组
- `confidence_distribution` — 0.35-0.5/0.5-0.7/0.7-1.0三档
- `status_distribution` — 按状态分组

### 4.6 参数配置（2个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `GET /api/settings` | 读取系统参数 | 是 |
| `PUT /api/settings` | 保存系统参数 | 是 |

参数校验范围：`detect_confidence: 0.1-1.0`, `downward_ratio: 0.5-1.0`, `min_vertical_distance: 10-500`, `min_track_frames: 1-100`, `roi_required_ratio: 0-1.0`, `alarm_cooldown_seconds: 0-300`

### 4.7 文件访问（1个）

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `GET /api/files?path=` | 文件访问 | 是 |

安全规则：
- 白名单目录：`uploads/`、`outputs/`、`events/snapshots/`
- 禁止 `../` 路径穿越
- 禁止绝对路径
- `os.path.realpath` 解析后前缀校验
- 不存在返回404，非法路径返回400

### 4.8 摄像头（4个）— 辅演示

| 接口 | 说明 | 鉴权 |
|---|---|---|
| `POST /api/camera/start` | 启动摄像头 | 是 |
| `POST /api/camera/stop` | 停止摄像头 | 是 |
| `GET /api/camera/status` | 查询摄像头状态 | 是 |
| `GET /api/camera/stream` | MJPEG实时流 | 否 |

MJPEG实现：`StreamingResponse` + `multipart/x-mixed-replace` + 每帧 `cv2.imencode('.jpg')`  
冲突处理：被占用返回503，重复启动不崩溃

## 五、接口与算法Pipeline集成

调用方式：
```python
from algorithm.pipeline import run_video_analysis

result = run_video_analysis(
    video_path="uploads/demo.mp4",
    output_dir="outputs/",
    roi={"x": 100, "y": 50, "width": 400, "height": 300},
    settings={"detect_confidence": 0.35, ...}
)
```

算法返回处理规则：

| 算法输出 | 后端处理 |
|---|---|
| `status=success` | video_tasks.status→success |
| `status=failed` | video_tasks.status→failed, error_message写入 |
| `events[]` | 批量INSERT events表 |
| `detection_results[]` | 批量INSERT detection_results表 |
| `tracking_results[]` | 批量INSERT tracking_results表 |
| `result_video_path` | 写入video_tasks |
| `total_frames/processed_frames` | 更新video_tasks |
